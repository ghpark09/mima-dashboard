## 1. 목적 & 목표

여러 구글 계정/워크북에 흩어진 **광고비 RAW**와 **매출 RAW**를 한곳에 모아, 매일 오전 10시에 자동 갱신되는 **단일 웹 대시보드**로 제공한다. URL만 있으면 PC·모바일 어디서나 접속 가능.

성공 기준:
- 매일 오전 10시(KST) 전일 데이터가 자동 반영된다.
- 실제 판매 매출(판촉 제외)과 매체별 광고 효율을 한 화면에서 비교할 수 있다.
- 월·매체 단위로 데이터를 필터링해 Excel로 한 번에 추출할 수 있다.
- 매일 자동으로 "요약 성과 + 개선 제안"이 제시된다.

## 2. 전역 표시 규칙 (모든 화면 공통)

1. **ROAS는 퍼센트로 표기** — `ROAS = 매출 ÷ 광고비 × 100`, 예: `590%` (절대 `5.9x` 아님).
2. **숫자 약어 금지** — `₩12,418,000` (절대 `₩12.4M`, `684K` 아님). 천단위 콤마 + ₩.
3. **매출 기준 = VAT 제외 순매출**(매출시트 `판매가(-vat)`, col U)을 기본값으로. 화면 토글로 VAT 포함(`판매가`, col I) 전환 가능.
4. **매출 정의 (뷰별로 다름)**:
   - **종합 요약/홈, 판매처별, 추세** → **실제 판매시트 매출**(판매구분=판매만, VAT 제외 순매출).
   - **매체 광고 효율, 월별 매체 리포트** → **매체 광고 전환매출**(네이버 총 전환매출액, 카카오 구매금액(7일), 메타 ROAS×광고비).
   - 두 매출은 **출처가 다르므로 절대 더하지 않는다**(따로 표시).

## 3. 아키텍처 (데이터 파이프라인)

```
[5개 시트] --(서비스계정/뷰어)--> Vercel Cron(매일 10:00 KST)
   → Google Sheets API로 읽기
   → 정제·집계 (정규화/판촉제외/매체매핑/일·주·월 집계)
   → 집계 결과를 캐시에 저장 (Vercel Blob 또는 KV, JSON)
   → 대시보드(Next.js)는 캐시만 읽음 → 항상 빠름, 10시에 갱신
```

- **추천: Vercel Cron + 서비스계정 + 집계 캐시.** 화면이 시트를 직접 호출하지 않아 빠르고 안정적.
- 갱신 주기: 매일 **10:00 KST**(Vercel Cron은 UTC 기준이므로 `0 1 * * *` = UTC 01:00 = KST 10:00).
- 수동 갱신 버튼(관리자용 `/api/refresh` 트리거)도 제공.

## 4. 권한 & 보안

- **편집 권한 불필요. 각 시트에 서비스계정 이메일을 "뷰어"로 공유**하면 됨.
- 구글 클라우드 콘솔에서 서비스계정 + JSON 키 발급 → 키는 Vercel **환경변수**로만 보관(레포에 커밋 금지).
- 매출/광고비는 민감정보 → 시트 "웹에 게시"는 사용하지 않음(서비스계정 방식이 비공개 유지).
- (선택) 대시보드 URL 자체에 간단한 비밀번호/접근 제한을 둘지 추후 결정.

## 6. 데이터 정제 (전수 분석에서 확인된 함정 반영)

### 매출시트
1. **헤더가 3행** — 1~2행(`#N/A`, `수식`)은 스킵.
2. **헤더 비어있는 열을 위치로 매핑**: A(id), H(수량), I(판매가), P(주문일복사), Q(연), R(월), S(일), T(주차), U(판매가-vat).
3. **판촉 4,513건 제외** — `판매구분='판매'`만 매출 집계. (col U는 판촉이 이미 `0`이라 합산해도 안전)
4. **금액 텍스트 정리** — 콤마 제거, col U는 앞뒤 공백도 제거 후 숫자화.
5. **VAT**: col I=VAT포함, col U=VAT제외(≈×0.989). 기본 col U.
6. **판매처 ≠ 채널구분1** — 분석은 `채널 구분1`(정규화) 사용, 원천 보기엔 `판매처`.
7. **모델명2 중복 정리**: `선데이 베이지`=`선데이베이지`.
8. **SKU(col J)** `#N/A`·공백 63+102건 → "미분류" 버킷.
9. **사장 열 제거**: `자리맥스용`(K), 끝열(W).

### 광고시트 공통
10. **카카오 날짜** `2026.05.27` → `2026-05-27` 정규화(안 하면 조인에서 카카오가 통째로 누락).
11. **카카오 헤더 오염**: col 0 `보고서명 : ...` → `캠페인 이름` 으로 하드매핑.
12. **네이버 빈칸 ≠ 0**: 브랜드검색/파워컨텐츠 행의 비용/전환은 "미보고" 처리(0 나눗셈 방지).
13. **메타 광고세트 예산은 지표 아님**(행마다 반복) → 합산 금지, campaign-day당 MAX. 메타(네이버)는 숫자+텍스트(`광고 세트 예산 사용 중`) 혼합 → 조건부 파싱.
14. **비율 컬럼(CTR/CPC/CPM/ROAS/빈도/당비용)은 합산 금지** — 집계 후 분자/분모로 재계산.
15. **CTR/ROAS는 이미 % 크기**(0.198=0.198%, 759.404=759%) — ×100 재차 금지.
16. **도달(reach)은 비가산** — 행/일/매체 간 단순합 금지(매체별만 표시).
17. **기여기간**: 1일/7일 혼합 금지, **7일 기준 일관 사용**.
18. **메타(네이버)는 메타 데이터** — 네이버 합계에 더하지 않음. 두 메타 계정은 "메타(전체)"로 묶고 계정별 하위분리.
19. **전부 0인 지표 컬럼**(회원가입/앱설치 등 카카오 99열 다수) → 숨김/옵트인.

## 7. 데이터 모델 (통합)

### 7.1 차원 (Dimensions)
- **시간**: 일(YYYY-MM-DD, 마스터 조인키), 주차, 월, 연. 매출시트는 연/월/일/주차 사전계산 사용, 광고시트는 날짜에서 파생.
- **판매**: 판매처(31), 채널구분1(~25, 분석용), 채널구분2(온/오프), 판매구분(판매/판촉), 상품명(106), SKU(65), 모델명(~10), 모델명2(~13).
- **광고**: 매체(카카오/네이버/메타미마/메타네이버 — 시트별 파생 컬럼), 캠페인이름(파싱: 집행구분 라이브/기획, 제품토큰), 유형/분류(매체별 상이), 목표(방문/전환), 메타 플랫폼/노출위치/기기.
- **매체 계층(드릴다운 축)** — 매체를 단일 행이 아니라 하위 단위로 펼칠 수 있게 한다:
  - **네이버**: 네이버 → `광고 구분`(검색광고 / 디스플레이광고) → `캠페인 분류`(브랜드검색·파워링크·쇼핑검색·파워컨텐츠·ADVoost·카탈로그 등 ~9) → 캠페인.
  - **카카오**: 카카오 → `캠페인 유형`(디스플레이 / 비즈보드) → 캠페인.
  - **메타**: 메타(전체) → 계정(미마 / 네이버) → `플랫폼`/`노출 위치` → 캠페인.

### 7.2 지표 (Metrics) — R=원천, D=파생
- **매출(판매시트)**: 매출(VAT제외, col U, **기본**)·매출(VAT포함, col I)·수량·주문수(판매 행수)·AOV(D).
- **광고**: 광고비(R: 카카오 비용/네이버 총비용/메타 지출금액), 노출·클릭·도달(R), CTR·CPC·CPM(D), 구매전환·구매금액(R, 희박), CPA(D), ROAS(R/D, **%표기**), Blended ROAS(D: 판매매출÷총광고비×100).
- **집계 규칙**: 모든 비율은 합산된 분자/분모로 재계산. 도달·메타예산은 합산 금지.

### 7.3 매체 ↔ 채널 매핑 (공통키 없음 → 명시 테이블)
| 매체 | 매핑 채널 | 신뢰도 |
|------|-----------|--------|
| 카카오 | 카카오(채널구분1) | 중 |
| 네이버 | 네이버 스토어=스마트스토어 | 높음 |
| 메타(미마)/(네이버) | 메타 전체(특정 채널 매핑 불가) | 낮음(추정) |

- **기간 겹침 주의**: 매출 1~6월 vs 광고 5월~6월초. **Blended/매체효율은 5/1~6/4 겹침구간만 유효**. 1~4월은 매출 단독 표시, 광고지표는 비활성.

## 8. 화면 구성 (7개 탭)

### 탭 1 — 종합 요약 (홈) + 일별 개선 제안
- 헤더 KPI(실제 판매 매출 기준): 어제 매출(순)·어제 광고비(합)·Blended ROAS%·주문수·AOV, 전일대비.
- 매체별 한줄 요약 카드(광고비/전환매출/ROAS%).
- **일별 개선 방안 자동 제안(인사이트 엔진)** — §9.

### 탭 2 — 일별 성과
- 일자별 매출 vs 광고비 이중축, 일별 Blended ROAS% 라인(무지출일은 끊김), 매체별 광고비 스택, 일별 주문/수량/판촉 테이블, 퍼널(노출→클릭→구매).

### 탭 3 — 매체 광고 효율 (전환매출 기준)
- 매체별 효율표(광고비·노출·클릭·CTR·CPC·CPM·도달·구매·CPA·전환매출·ROAS%), ROAS% 막대, 캠페인 드릴다운, CTR×CPC 산점도, 목표별(방문/전환), 메타 지면×기기 히트맵.
- **매체 행 확장(드릴다운)**: 네이버 → 검색광고/디스플레이광고 → 캠페인 분류, 카카오 → 디스플레이/비즈보드, 메타 → 계정별. 네이버의 검색 vs 디스플레이 효율을 별도 매체처럼 비교 가능.

### 탭 4 — 판매처별 매출 (실제 판매 매출)
- 판매처/채널구분1 랭킹, 온·오프라인 분할, 판매vs판촉 구성, 모델명×채널 히트맵, 상품 Top 테이블, 변형(모델명2) 분해.

### 탭 5 — 추세·누적 (실제 판매 매출)
- 월별/주별 매출, 누적 YTD(area), 누적 광고비 vs 누적 매출(겹침구간), 누적 Blended ROAS%, 전월대비(MoM) 테이블, 주별 채널믹스(stacked area), 누적 수량.

### 탭 6 — 월별 매체 리포트 (요청 기능)
- **필터**: 월(1~6, 다중) · 매체 · 일/월 단위 토글 · **매체 세분화 토글**(네이버 검색/디스플레이 등 하위 펼침).
- **출력 피벗**: 행=매체(+합계), 열=총광고비·전환매출·ROAS%·노출·클릭·CTR·구매·CPC.
- **매체 행 확장**: 네이버 → 검색광고/디스플레이광고(필요 시 캠페인 분류까지), 카카오 → 디스플레이/비즈보드, 메타 → 계정별. 펼침/접힘으로 요약과 세부를 전환.
- **일단위 토글** 시 날짜별로 펼침(일별 데이터).
- **[Excel(.xlsx) 다운로드]** — 선택한 월·매체·세분화 수준 그대로 한 번에 추출(서식·합계 포함).

### 탭 7 — 채널별 / 캠페인별 분석 (신규)
전 매체를 가로질러 캠페인 단위까지 파고드는 전용 분석 탭.
- **계층 그룹핑 토글**: 매체 → 채널/유형(네이버 검색·디스플레이 / 카카오 디스플레이·비즈보드 / 메타 플랫폼·지면) → 캠페인. 원하는 레벨에서 롤업/드릴다운.
- **정렬·필터 가능한 캠페인 테이블**: 캠페인 | 매체 | 채널/유형 | 집행구분(라이브/기획) | 광고비 | 노출 | 클릭 | CTR% | CPC | 구매 | CPA | 전환매출 | ROAS%. 모든 열 정렬, 매체·유형·집행구분·기간 필터.
- **캠페인 ROAS% 랭킹**(상위/하위), **광고비 점유 트리맵**(채널/캠페인 비중), **선택 캠페인 일별 추이**(광고비·ROAS%).
- 채널 단위 비교: 같은 매체 내 채널/유형별(예: 네이버 파워링크 vs 쇼핑검색 vs 브랜드검색) 효율 한눈에.

## 9. 인사이트 엔진 (일별 개선 방안 제안)

**1차: 규칙 기반(rule-based)** — 매일 집계 후 자동 생성.
- ROAS% 임계치 미달 매체 → "예산 축소/재배분 검토"
- 광고비 발생·전환 0 캠페인 → "일시중지 검토"
- CTR 전일/평균 대비 급락 캠페인 → "소재 교체 검토"
- 최고 효율(ROAS 상위) 캠페인 → "예산 증액 기회"
- 특정 판매처 매출 급증/급감(±임계%) → 알림
- 임계치는 설정값(config)로 조정 가능.

**2차(선택): AI 내러티브** — 위 규칙 결과 + 핵심 수치를 Claude API로 보내 자연어 일일 브리핑 생성(옵션, 추후).

## 11. 비범위 (Non-goals) / 제약

- 전년대비(YoY) — 2026년 단일 연도라 불가.
- 광고→상품 단위 정밀 귀속 — 캠페인명 토큰 기반 추정만(저신뢰, "추정" 라벨).
- 매체간 총도달 합산 — 비가산이라 미제공.
- 광고 1~4월 데이터 없음 — 해당 구간 광고지표 비활성.
- 실시간(초단위) 아님 — 일 1회(10시) 갱신.

## 12. 확인 필요 / 가정

- 광고비의 VAT 포함 여부는 매체별로 불명확(메타/네이버 통상 VAT 제외 청구) — 일단 "매체 청구 기준"으로 표기, 정산 대조 시 재확인.
- 메타(미마)는 전환매출 컬럼이 없어 ROAS×광고비로 역산 → 값이 희박/부정확할 수 있음.
- 대시보드 접근 제한(비밀번호) 필요 여부.
- 인사이트 임계치 기본값(ROAS 목표선 등) 구체 수치.
